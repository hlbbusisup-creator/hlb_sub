# HLB 현장지원팀 · 사업지원 대시보드

HLB 그룹 계열사들의 **사업 ↔ 이사회 ↔ 현장지원팀 활동**을 하나의 지식그래프(마인드맵)로
연결해 한눈에 보는 내부용 대시보드입니다.

> 현장지원팀이 여러 계열사 사업을 동시에 지원하다 보면 "지금 누가 어느 회사의 무슨 건을
> 돕고 있는지"가 흩어집니다. 이 도구는 그 관계를 그래프로 묶어, 이후 업무로그·주간보고까지
> 자동으로 얹을 수 있는 토대를 만듭니다.

## 전체 구상 (3단계)

| 단계 | 내용 | 상태 |
| --- | --- | --- |
| **1. 지식그래프** | 계열사·사업·이사회·팀원·지원활동을 노드로 연결한 마인드맵 | ✅ v1 (이 버전) |
| 2. 업무로그 | 팀원 5명이 가볍게 남기는 일일 업무 입력 → 그래프에 자동 연결 | 예정 |
| 3. 주간보고 | 누적된 로그·활동을 AI가 모아 주간보고 자동 생성 | 예정 |

## v1에 들어있는 것

- **노드 5종**: 계열사 · 사업 · 이사회 · 현장지원팀원 · 지원활동
- **자동 레이아웃 그래프** (dagre 좌→우 계층): 노드 클릭 시 연결 강조 + 우측 상세 패널
- **계열사 필터**: 특정 회사 기준으로 그래프/지표 좁혀보기
- **KPI 바**: 계열사 수, 진행 사업, 진행 지원활동, 이사회 수

## 데이터 모델

```
계열사(company) ──보유──→ 사업(business)
계열사(company) ──개최──→ 이사회(board) ──논의/의결──→ 사업(business)
팀원(member)   ──수행──→ 지원활동(support) ──지원──→ 사업/계열사
```

정의: `src/data/types.ts` · 데이터: `src/data/seed.ts`

> ⚠️ `seed.ts`의 계열사명은 공개된 그룹사명이지만, 사업·이사회 안건·지원활동 내용은
> 구조 시연용 **예시(가상)** 입니다. 실제 운영 시 이 파일을 팀 실데이터로 교체하세요.
> 비상장사 이사회 내용은 민감정보이므로, 다음 단계에서 저장 위치·접근권한을 먼저 설계합니다.

## 실행

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # 타입체크 + 프로덕션 빌드
```

## 기술 스택

React + TypeScript + Vite, 그래프 렌더링 `@xyflow/react`, 레이아웃 `dagre`.
v1은 백엔드 없이 프론트엔드 단독으로 동작합니다(데이터는 `seed.ts`). 2단계에서
입력·저장이 필요해지면 백엔드/DB를 도입합니다.
